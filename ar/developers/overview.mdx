تتيح لك **واجهة Integration API من فاتورلي** أن تُنشئ فواتيرك الإلكترونية وترسلها
عبر شبكة Peppol برمجيًا من أنظمتك الخاصة — سواء كان نظام تخطيط موارد المؤسسات
(ERP) أو منصة فوترة أو نظامًا خلفيًا مخصّصًا. وهي **مكمّلة لتطبيق الويب**: كل ما
يمكنك فعله في محرّر الفواتير يمكنك فعله أيضًا عبر الواجهة، فتؤتمت العمليات عالية
الحجم أو المكتبية مع استمرار استخدامك لواجهة المنصة للأعمال الفردية.

## العنوان الأساسي

تُوجَّه جميع الطلبات إلى العنوان الأساسي المُضمَّن للإصدار:

```
https://integration.fatorly.com/v1
```

## الإصدارات

الإصدار الحالي هو **v1**، ويُحمَل في مسار العنوان (`/v1`). يُبقي تثبيتك على إصدار
معيّن تكاملك مستقرًا مع تطوّر المنصة.

<Note>
الواجهة **صادرة فقط** (Outbound). أنت تُنشئ المستندات التي تُصدرها وترسلها (الفواتير
الضريبية وإشعارات الخصم). أما المستندات **الواردة** فتصل عبر Peppol ولا يمكن
إنشاؤها عبر الواجهة.
</Note>

## ما الذي يمكنك فعله

- إنشاء فواتير ضريبية وإشعارات خصم عبر `POST /invoices`.
- إدارة البيانات المرجعية التي تُصدر الفواتير مقابلها — العملاء والأصناف.
- البحث عن المشاركين في شبكة Peppol.

كل عملية كتابة محميّة بترويسة **`Idempotency-Key`** كي لا تُنشئ إعادات المحاولة
نسخًا مكرّرة — راجع [معرّف عدم التكرار](/ar/developers/idempotency).
